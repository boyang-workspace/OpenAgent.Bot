# OpenAgent.bot Blog Standard

Future OpenAgent.bot blog posts should match the quality bar of a serious SEO comparison guide, not a short editorial note. The reference standard is an article like `https://www.omniscriber.com/blog/ai-chat-export-tools-comparison-1776031257903`: search-intent first, comparison-heavy, practical, and conversion-aware.

## Quality Bar

Every publishable blog post should:

- Answer the target search query in the first 100 words.
- Be useful to a human reader without requiring them to open every source link.
- Include concrete examples, steps, tradeoffs, or comparison criteria.
- Link naturally to relevant OpenAgent resource pages, category pages, and source pages.
- Separate facts from editorial interpretation.
- Avoid thin launch-summary writing, generic praise, and repeated claims.

## Recommended Structure

Use this default structure unless the topic clearly needs a different format:

1. **Title**
   - Clear search intent.
   - Prefer comparison, guide, or decision phrasing.
   - Example: `Best Open-Source Browser Agents for Real Workflow Automation`.

2. **Intro**
   - 2-4 short paragraphs.
   - Name the pain, answer the query, and preview the recommendation.

3. **Quick Recommendation**
   - Best overall.
   - Best for developers.
   - Best for local/self-hosted use.
   - Best for beginners.

4. **Comparison Table**
   - Columns should include tool/resource, best for, open-source status, setup difficulty, key limitation, and next step.

5. **Deep Dives**
   - One section per tool/resource.
   - Cover what it is, when to use it, when not to use it, and official links.

6. **How To Choose**
   - Translate features into decision criteria.
   - Include practical tradeoffs, not only tags.

7. **Workflow Or Steps**
   - Give the reader a concrete next action.
   - For OpenAgent topics, include install/clone commands only when verified from official sources.

8. **OpenAgent CTA**
   - Link to the relevant resource page, category page, or submit page.
   - The CTA should feel editorial, not salesy.

9. **FAQ**
   - 4-6 questions based on likely search intent and People Also Ask style queries.

## Minimum Requirements

A blog post is not ready to publish unless it has:

- A clear primary keyword or search intent.
- At least 1 comparison table or structured comparison block.
- At least 3 concrete examples, tools, or resources.
- At least 3 internal links to OpenAgent pages when relevant.
- At least 3 official external source links.
- A meta title and meta description.
- A FAQ section.
- A direct next step for the reader.

## Voice

Use a calm, direct, practical tone:

- Prefer plain-English explanations.
- Use short paragraphs.
- Make tradeoffs explicit.
- Do not hype projects.
- Do not overclaim license, benchmark, or capability facts.

## OpenAgent-Specific Defaults

For OpenAgent.bot, the strongest blog formats are:

- `Best X open-source tools for Y`
- `X vs Y for agent builders`
- `How to choose an open-source AI memory system`
- `What is X, and when should builders care?`
- `Open-source alternatives to X`

Blog articles should support the directory. They should link back into:

- `/models`
- `/agents`
- `/skills`
- `/memory-systems`
- Specific resource pages such as `/agents/openclaw`

## Publish Gate

Before publishing, review the draft and reject it if:

- It reads like a compressed README.
- It lacks comparisons.
- It has no actionable next step.
- It does not link to relevant OpenAgent pages.
- It does not include source links.
- It makes unsupported open-source, license, benchmark, or capability claims.
